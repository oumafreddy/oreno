API Documentation

RESTful API endpoints for integrating with the Oreno GRC platform

API Overview

The Oreno GRC API provides programmatic access to all platform features, enabling seamless integration with external systems and custom applications.

Base URL

https://oreno.tech/api/

Authentication

JWT Bearer tokens for secure API access

Format

JSON request/response format

Authentication

All API requests require authentication using JWT Bearer tokens. Obtain your token by logging in through the web interface.

Example Request

curl -H "Authorization: Bearer YOUR_JWT_TOKEN" \
     -H "Content-Type: application/json" \
     https://oreno.tech/api/audit/engagements/

API Endpoints

Audit Management

GET /api/audit/engagements/

Retrieve list of audit engagements

Response: Array of engagement objects with metadata
POST /api/audit/engagements/

Create a new audit engagement

Body: Engagement object with title, objectives, and timeline
GET /api/audit/issues/

Retrieve list of audit issues

Response: Array of issue objects with risk assessments

Risk Management

GET /api/risk/risks/

Retrieve list of risks from risk register

Response: Array of risk objects with assessments and treatments
POST /api/risk/risks/

Create a new risk entry

Body: Risk object with name, description, and assessment data

Compliance Management

GET /api/compliance/requirements/

Retrieve compliance requirements

Response: Array of compliance requirement objects

Error Handling

The API uses standard HTTP status codes to indicate success or failure of requests.

400 Bad Request

Invalid request parameters or malformed JSON

401 Unauthorized

Missing or invalid authentication token

403 Forbidden

Insufficient permissions for the requested resource

404 Not Found

Requested resource does not exist

500 Server Error

Internal server error occurred
